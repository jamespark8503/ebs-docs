DocAI Kit 설치 방법 (Online / Offline)

Online: pip 설치

# 레포 클론 후 설치 (python >= 3.10) git clone git@github.com:UpstageAI/enterprise-docai-kit.git cd enterprise-docai-kit pip install -r requirements.txt

Offline: Docker 이미지

# Docker 이미지 로드 (USB로 반입) docker load -i docai_kit_v1.1.0_20251105.tar.gz # 컨테이너 실행 docker run -it \ -p 30050:8080 \ -v {workspace_path}:/workspace \ docai-kit:1.1.0

Docker 이미지 경로: /data/project/enterprise/enterprise-docai-kit/docker/
최신 버전: docai_kit_v1.1.0_20251105.tar.gz (v1.1.0 — sort, 영역 조절, 엑셀 추출 기능 추가)
1
워크스페이스 생성
프로젝트 디렉토리 구조 세팅
# 디렉토리 생성 mkdir -p {workspace}/project/{document_type}/images mkdir -p {workspace}/project/{document_type}/outputs
{workspace}/
├── project/
│   └── {project_name}/          ← 폴더명 = Validator 문서 이름
│       ├── images/               ← 추론할 이미지 넣기
│       └── outputs/              ← 결과 자동 생성
├── pack/                         ← Online에서 모델 패키지 마운트 시
│   ├── ocr/
│   └── parser/{document_type}/
├── validator/                    ← set.sh 실행 후 자동 생성
└── setup.sh                     ← 환경변수 설정 (필수)
# 이미지 복사 예시 cp /data/project/datasets/.../*.png {workspace}/project/{project_name}/images/
⚠️
project_name = Validator에 표시되는 문서명. 폴더명을 신중하게 정하세요.
parser_document_types에서 이 폴더명을 실제 document_type으로 매핑합니다.
2
setup.sh 설정
환경변수 설정 (포트, 엔진, 매핑)
# setup.sh 핵심 구조 workspace="/workspace" ip=172.16.201.12 # 서버 IP # OCR ocr_port="9004" # Parser 포트 (문서타입별) parser_ports='{ "car_dashboard": "30000" }' # Parser 엔진 (drsp 또는 kvs) parser_engines='{ "car_dashboard": "kvs" }' # 프로젝트 폴더명 → 문서타입 매핑 parser_document_types='{ "cd": "car_dashboard" }' # ↑ 왼쪽: project 폴더명, 오른쪽: 실제 document_type # Serving Pack (고객사 환경) controller_port="8394" serving_models='{ "cd": "car_dashboard" }' # ↑ key: 폴더명, value: controller에 등록된 모델명
CategoryVariableDescription
공통workspace워크스페이스 경로
ip서버 IP (Online: 서버IP / Offline: Serving Pack IP)
OCRocr_portOCR 컨테이너 포트
Parserparser_ports"document_type": "port" (JSON)
parser_engines"document_type": "drsp" 또는 "kvs" (JSON)
parser_document_types"폴더명": "document_type" (JSON) — 가장 중요!
Servingcontroller_portServing Pack 포트 (Offline 전용)
serving_models"폴더명": "모델명" (JSON)
# 환경변수 로드 (컨테이너 내부에서) source /workspace/setup.sh
🚨
초기 실행 오류의 80%는 setup.sh 오타입니다.
- JSON key 끝에 s 확인 (parser_ports, parser_engines)
- 콤마(,) 빠뜨림 또는 마지막 항목 뒤에 콤마 확인
- setup.sh는 /workspace에 백업 권장 (컨테이너 내려가면 사라짐)
3
Docker 컨테이너 실행
OCR + Parser 모델 서빙 (Online 전용)
📌
Offline(고객사)에서는 Serving Pack이 이미 서빙 중이므로 이 단계를 건너뛰고 Step 4로 이동합니다.

GPU 노드 접속

srun --gres=gpu:1 --cpus-per-task=4 --mem=128G --partition=normal --pty bash -i # 컨테이너 모니터링 sudo watch -n 2 docker ps

OCR 컨테이너

# 이미지 로드 sudo docker load -i /data/project/docker/images/ocr-2.5.0.tar.gz # 실행 sudo docker run --rm -d \ --gpus '"device=0"' \ --shm-size 64G \ -p 9004:8080 \ -v {OCR_MODEL_DIR}:/model \ -v {OCR_CONFIG_DIR}:/config_dir \ -e OP_CONFIG_DIR=/config_dir \ -e OP_MODEL_DIR=/model \ ocr:2.5.0-kserve

DRSP Parser 컨테이너 (3세대)

# 이미지 로드 sudo docker load -i /data/project/enterprise/docai/docker/images/docai3/{nlp_engine_image}.tar.gz # 실행 sudo docker run --rm -d \ --gpus all --ipc=host \ -v {package_path}/{document_type}:/pack \ -e OP_MODEL_DIR=/pack/{model_dir} \ -e OP_CONFIG_DIR=/pack/{config_dir} \ -p 30000:8080 \ nlp-engine:{tag}
📌
DRSP vs KVS 차이: DRSP는 OP_CONFIG_DIR 설정이 필요하며, UFO(텍스트)만으로 파싱합니다. 이미지 경로: /data/project/enterprise/docai/docker/images/docai3/

KVS Parser 컨테이너 (4세대)

# 이미지 로드 sudo docker load -i /data/project/enterprise/docai/docker/images/docai4/{kvs_image}.tar.gz # 실행 sudo docker run --rm -d \ --gpus device=1 --ipc=host \ -v {package_path}:/pack \ -e OP_MODEL_DIR=/pack/weight \ -p 30000:8080 \ kvs-engine:{tag}

DocAI Kit 컨테이너

sudo docker run -it \ -p 30050:8080 \ -v {workspace}:/workspace \ docai-kit:1.1.0
⚠️
CSV/XLSX 출력이 필요하면 Parser config 디렉토리를 추가 마운트해야 합니다 (ontology_short.yaml 필요):
-v {package_path}/config:/workspace/pack/parser/{document_type}/config

# 기본 추론 (Validator용) bash scripts/e2e/ocr2parser.sh # CSV/XLSX 출력 포함 (BMT 결과 제출용) bash scripts/e2e/ocr2res.sh

Offline: Serving Pack

# Serving Pack에 요청 (고객사 환경) bash scripts/e2e/serving2all.sh

E2E 스크립트 선택 가이드

스크립트용도출력
ocr2parser.sh기본 추론 (Validator용)ufos.json + mufps.json
ocr2res.shBMT 결과 제출용+ results.csv
serving2all.shServing Pack 호출 (Offline)oac.json
ocr2all.shOCR만 실행ufos.json
conversion2all.shPDF/TIFF 변환 + OA만PNG 이미지
💡
오류 시: ocr2all.sh부터 단계별로 실행해보세요. 대부분 setup.sh의 포트/IP 오타입니다.
OCR 이미 완료 시: ocr2parser.sh는 내부적으로 ufos.json 존재 여부를 확인해서 OCR을 스킵합니다.
5
Validator (웹 채점)
추론 결과 시각화 + 채점
⚠️
Offline(고객사)에서 Validator를 사용하려면 포트가 열려있어야 합니다.
정기점검 전 반드시 포트 신청 (예: 30050포트 오픈 요청)

5.1 output_format 설정

scripts/validator/set.sh에서 output_format을 설정합니다:

환경output_format사용 파일
Online (ocr2parser)"mufp"mufps.json
Offline (serving2all)"oac"oac.json

5.2 Validator 실행

# 1. vufp 포맷으로 변환 bash scripts/validator/set.sh # 2. Validator 웹서버 실행 bash scripts/validator/run.sh # 브라우저에서 접속 # http://{서버IP}:{포트번호}

5.3 채점 결과 JSON 추출

# Validator가 실행중인 상태에서 curl로 점수 추출 curl -s http://localhost:{port}/score/{project_name} \ -o /workspace/project/{project_name}/outputs/score_output.json
6
결과 변환 (CSV/XLSX)
BMT 결과 제출용 파일 생성

ocr2res.sh를 사용하면 CSV가 자동 생성됩니다. XLSX가 필요하면:

# XLSX 출력 (컨테이너 내부에서) pip install openpyxl # ocr2res.sh 맨 끝에 추가 (vi로 편집) python code/ufp2csv.py --outputs $outputs --ontology $ontology_path --output_format "xlsx"

고객사 정답 파일과 비교

# 고객사 정답 CSV와 예측 결과를 문서명 기준으로 merge python -c " import pandas as pd pred = pd.read_csv('/workspace/project/{name}/outputs/results.csv') answer = pd.read_csv('/workspace/client_answer.csv') merged = answer.merge(pred, on='파일명', suffixes=('_정답','_예측값')) merged.to_csv('/workspace/merged_result.csv', index=False) print('Saved merged_result.csv') "
⚠️
컨테이너를 내리면 수정사항이 사라집니다. /workspace에 저장된 파일만 유지됩니다. 스크립트 수정은 Docker 이미지를 다시 빌드해야 영구 반영됩니다.
