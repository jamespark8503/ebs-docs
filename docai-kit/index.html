Core Architecture Pattern

모든 모듈이 동일한 3-레이어 구조를 따릅니다.

request_XXX()
→
단일 이미지/문서에 대한 HTTP 요청 (서빙 서버 통신)
run_XXX()
→
디렉토리 내 전체 파일 순회하며 request_XXX() 호출
main()
→
argparse CLI 인자 처리 후 run_XXX() 실행 + JSON 저장

Module Details
각 모듈의 역할과 핵심 함수
📄
conversion.py
PDF/TIFF 등 멀티페이지 문서를 단일 PNG 이미지로 분리 변환합니다. ProcessPoolExecutor로 병렬 처리하며, 원본은 backup/ 폴더로 이동합니다.
file_processing() pdf_tiff_to_png_conversion()
🔄
oa.py
Orientation Alignment. Word box 각도 분석으로 문서 기울기를 감지하고 이미지를 자동 회전 보정합니다. 5도 미만 기울기는 스킵합니다.
apply_orientation_alignment() calc_global_angle() rotate_points() turn_image()
📑
classifier.py
CloudEvent 프로토콜로 이미지를 바이너리 전송하여 문서 타입을 분류합니다. 결과값(문서 타입명)으로 하위 폴더를 생성하고 이미지를 복사합니다.
request_classifier() run_classifier()
👁
ocr.py
이미지를 Base64 인코딩하여 OCR 서버에 전송합니다. withDOS, languageTag(기본: ko) 옵션을 지원하며 UFO 포맷으로 결과를 저장합니다.
request_ocr() run_ocr()
📚
parser.py
두 가지 엔진을 지원합니다. DRSP는 UFO만으로 텍스트 기반 파싱, KVS는 UFO+Image로 이미지+텍스트 기반 파싱을 수행합니다. 출력: UFP 또는 MUFP.
request_drsp() request_kvs() run_drsp() run_kvs() run_parser()
🤖
llm.py
Parser의 MUFP 결과를 LLM 서버에 전송하여 엔티티 값을 보정합니다. 문서 타입별로 별도 LLM 포트/모델 운용이 가능합니다.
request_llm() run_llm()
🚫
masker.py
UFP의 entity key 기준으로 민감정보를 선별하고, bounding box 좌표에 PIL ImageDraw로 검은색 사각형을 덮어 마스킹합니다.
extract_bbox_for_crop() image_masking() run_masker()
📊
ufp2csv.py
Ontology YAML에서 .content 키를 추출하여 컬럼을 생성하고, MUFP의 group → entity 구조를 flat 레코드로 변환하여 CSV 또는 XLSX로 저장합니다.
convert_ufp_to_record() save_csv() save_xlsx()
⚡
dp.py
Document Parsing 통합 모듈. OCR + Parser를 하나의 서버에서 처리합니다. 이미지만 입력하면 UFO 없이 직접 파싱 결과를 반환합니다.
request_dp() run_dp()
🚀
serving.py
Serving Pack 통합 추론. 다른 모듈과 달리 multipart form-data로 이미지 + 모델명을 전송합니다. 엔드포인트: /inference?verbose=True
request_inference() run_inference()
🔧
utils.py
공통 유틸리티. JSON/YAML 로드/저장, MIME 타입 감지, 멀티페이지 이미지 로딩 (PNG, JPEG, PDF, TIFF 지원)을 제공합니다.
save_json() load_json() load_yaml() load_multipage_images()
📝
mbr_convert.py
일회성 스크립트. MUFP의 그룹 엔티티 키를 재구성하고 항목 엔티티를 삽입하는 MBR(의료비) 전용 후처리입니다.
inline script
