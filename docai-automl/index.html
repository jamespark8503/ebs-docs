3세대 DSP vs 4세대 KVS
세대별 핵심 차이점 비교
🔸 3세대 DSP (DRSP)
  • UFO 텍스트 기반 파싱 (이미지 없이 가능)
  • UFO relations 기반 엔티티 추출 (parser.key_info, parser.group_info)
  • Prompt: ParallelNL (자연어 쿼리)
  • PostProcessor: Refiner + Grounder 중심
  • 단일 시퀀스 디코딩
  • Group 구조를 UFO relation에서 직접 파싱
  • ServingPipeline 클래스로 배포
🔵 4세대 KVS
  • Image + OCR 멀티모달 파싱 (이미지 필수)
  • End-to-end 학습 기반 엔티티 추출
  • Prompt: SlotFilling, Parallel, QA 3종 지원
  • PostProcessor + MultiPageMerger 추가
  • Parallel Decoding (다중 인스턴스 동시 디코딩)
  • DINOv2 Visual Encoder 옵션 추가
  • Temperature Scaling 기반 신뢰도 캘리브레이션

Core Modules

👁
SwinEncoder
이미지를 패치로 분할 후 계층적 특징 추출
Input: (B, 3, 1280, 960)
Output: (B, 1024, H, W)
Layers: (2, 2, 14, 2) / Window: 10
📝
BARTEncoder
OCR 텍스트 + 공간 정보(bbox)를 인코딩
Base: KoBART (hyunwoongko/kobart)
Hidden: 768-dim
Max tokens: 512
🔗
LXMERTEncoder
Visual + Textual 특징 간 Cross-Attention 수행
Fusion layers: 2
Strategy: Learnable feature replacement
Dropout: configurable
💬
MBARTDecoder
Auto-regressive하게 구조화된 출력 시퀀스 생성
Layers: 4 decoder layers
Max length: 1536 tokens
Generation: Greedy / Custom

Data Conversion Flow

# 1. UFO → UFP 변환 (convert_ufo_to_ufp.py)
ufo2gold_ufp(ufo) → UFP
  ├─ UFO relations에서 parser.key_info, parser.group_info 추출
  ├─ common.root 태그로 루트 관계 식별
  ├─ entity chain + group 구조 빌드
  └─ PostProcessor 적용 (Refiner + Grounder)

# 2. UFP → UFK 변환 (convert_ufp_to_ufk.py)
ufp2ufk(ufp, ontology) → UFK
  ├─ groups → flat key-value 리스트
  ├─ group_id 할당: {group_type}-{group_order} 또는 -1
  └─ ontology 매핑으로 group type 정보 추가

# 3. Dataset 로딩 (dataset.py)
ParsingDataset(images, ufos, ufks, ontology)
  ├─ 이미지 로드 + OCR(UFO) 로드 + 정답(UFK) 로드
  ├─ PromptManager로 학습용 prompt 생성
  ├─ 이미지 → tensor (grid mapping 포함)
  ├─ OCR → tokenized sequence + bbox
  └─ 배치 생성 (Collator로 패딩)

PostProcessing Pipeline

Module역할세부
Refiner값 정규화날짜/숫자/주소 포맷 정리. 문서타입별 규칙
GrounderDB 매칭추출값을 알려진 DB와 fuzzy matching 후 보정
MultiPageMerger멀티페이지 병합특수 토큰(<multi_page_entity_start/end>)으로 페이지 간 엔티티 병합

Inference Pipeline
추론 실행 흐름 및 서빙 구조
# test.py / infer.py 실행 흐름

1. Config 로드 → model_dir/config.yaml
2. Pipeline 초기화
   ├─ 모델 로드 (best / last / calibrated)
   ├─ Preprocessor + PromptManager 셋업
   └─ Logit Scaler 등록 (calibrated 모델)

3. 이미지별 추론
   ├─ preprocess() → collate() → tensors
   ├─ model.inference() → output_sequences + logits
   ├─ decode_prompt() → 구조화된 KV pairs
   ├─ confidence 계산 (calibrated)
   └─ UFP 포맷 변환

4. PostProcessing
   ├─ Refiner → 값 정규화
   ├─ Grounder → DB 매칭
   └─ MultiPageMerger → 페이지 병합

5. 출력
   ├─ pred.json (UFP)
   ├─ pred_mufp.json (MUFP)
   ├─ scores.txt (평가 결과)
   └─ result.tsv (예측 + 신뢰도)

5.1 체크포인트 저장

5.2 패키지 생성

$ ${docai3 또는 docai4} pack
? Select Document Type → 문서 타입
? Select Model → 모델
? Enter use case → use_case 입력
? Enter package name → kr-{type}-extractor-{use_case}-0.3b-{version}
패키지 이름 규칙: {language}-{document_type}-extractor-{use_case}-{size}-{gen}.{minor}.{patch}
예: kr-transfer-confirmation-extractor-meritzsec-0.3b-4.1.0
