일정

  • DevOps 팀 동반 출장 여부 조율 → 방문 일정 확정

출장 전 준비 — AIMP

기술 패키지

  • DocAI Kit 반입 패키지 빌드 · 사내 테스트
  • 모델별 후처리 로직 문서화 (내부 · 게이트웨이)

(반출 가능 시) 데이터 마스킹

  • 이미지 마스커 모델 반입 신청
  • JSON 마스커 코드 반입 신청

문서 · 교육

  • KPI 정의 & 산식 기록 (프로젝트 당시 Context 포함)
  • 고객사용 Validator 사용법 영상 제작 → PM 통해 배포

현장 운영

  • 작업 PC 위치 vs 현업 자리 확인 → 이동 지원(Dev/PM) 요청
  • 점검 대상 모델 및 도커 이미지가 떠 있는 서버에 접근 가능한 계정 사용 여부 체크
  • 해당 계정 docker 및 podman 사용 가능 여부 체크
  • (Validator 사용을 위한) 로컬 브라우저에서 연결 가능한 포트 신청
🚨
KPI Context 파악 필수! EBS와 협상한 것은 고객사 PM, 채점하는 것은 현업 → 둘 간의 이해도 간극이 있습니다. 프로젝트 수행 당시 Context를 반드시 사전에 기록해두세요.

Lessons Learned
실전 회고에서 뽑아낸 핵심 교훈
KB손해보험 회고
  • 방문 전 정기점검 가능 여부 확인이 최우선 (PM 통해 진행)
  • vufp가 재부팅 시 삭제되어 매번 재설치 필요
  • 현업 부서가 여러 개일 경우 부서별 대응 방안 사전 협의
  • 채점에 일주일 이상 소요될 수 있음 (11종 문서)
삼성증권 회고
  • 전체 프로세스 2~3주 소요 (validator 설치 1~2일 → 고객사 채점 2~3일 → 결과 확인/리포트 1~2일)
  • Validator 띄운 PC 방화벽 포트 미오픈 → 현업 화면에서 확인 불가 → 물리적 이동 필요
  • 50장 채점은 고객사 담당자에게도 부담 → KPI 대상 항목만 채점하도록 ontology-short 확인 필수
  • 개인정보 포함 이미지 반출 불가 → 현장에서 직접 확인해야 하는 번거로움
  • 진행 중인 프로젝트와 일정 겹칠 시 PM-Dev-Model 사전 협의 필요
전체 회고 — 핵심 교훈
  • 유지보수도 프로젝트 첫 세팅이 중요 — 당시 Context 기록이 없으면 KPI 기준부터 혼란
  • 개인기로 풀어낸 사이트(KB, HMM, AIA, 현대해상 등) → 당시 담당 Model/PM/Dev 없으면 유지보수 매우 어려움
  • 고객사 프로젝트 담당자 퇴사/이직 시 → 새로운 기준으로 채점 진행 가능 (히스토리 정리 필수)
  • Validator export 기능 부재 → 채점 결과 JSON 내려받기 기능 추가 개발 필요
  • 채점 기록 유실 사례 (새로고침, 새 파일 추가 등) → Validator에 자동 저장 기능 필요

정기점검 관련 고객사 소통 시 사용
정기점검을 위해 방문 가능한 일정과 점검 시 모델을 테스트할 테스트셋 준비를 요청드리고자 합니다. - 테스트셋 채점은 vufp로 진행하고자 하여, 사전 반입이 필요합니다. 일정 확인해주시면 감사하겠습니다. - 채점은 공정성을 위해 고객사 담당자분들께서 직접 하시는 방향으로 소통해주시면 감사하겠습니다. - 채점 후 vufp 결과 반출 가능 여부도 확인 부탁드립니다. [테스트셋 가이드] - 프로젝트 종료 시 협의된 데이터 분포와 유사한 일반적인 데이터로 준비 (틀린 케이스만 모은 것 X) - 수기 채점을 고려하여 모델당 최대 50장 준비 요청 (edge case 검토 후 30~50장으로 픽스 예정)
