환경 세팅
Conda 환경 + docai-automl 설치

Option A) 개인 가상환경 생성

$ source ${개인 경로}/bin/activate
$ conda create -n ${환경이름} python=3.10
$ pip install git+https://github.com/UpstageAI/docai-automl.git@main

Option B) 공용 가상환경 사용

$ source /data/project/enterprise/docai/miniconda3/bin/activate docai-automl
1
데이터 준비
Dataset 추가 → 온톨로지 → OCR 실행

1.1 데이터셋 & 온톨로지 경로 확인

항목경로
데이터셋/data/project/datasets/data_assets/{dataset_type}/{dataset_name}
온톨로지/data/project/enterprise/docai/repos/ocr-parser-ontology/_composed/{ontology_name}
💡
온톨로지가 없으면: cd .../ocr-parser-ontology/_composed && git checkout composed && git pull

1.2 데이터 추가

# Command 방식 (한 줄 실행) ${docai3 or docai4} data add \ -d payment_confirmation \ -n 250213_meritzsec_main \ -src /data/project/datasets/data_assets/payment_confirmation/drp.en_ko.in_house.deepnatural \ -ver 230525 \ -ont ont.payment_confirmation.upstage/4.0.yaml

또는 Prompt 방식 (대화형 CLI):

? Select Document Type (화살표로 선택)
  » consent_to_collection_and_use_of_personal_information
    Create New Directory

? Select Add Type
  » ontology — 온톨로지 추가 시
    source — 데이터셋 추가 시
데이터셋 이름 컨벤션: {날짜}_{고객사코드}_{범위}_{특징}
예: 250213_meritzsec_main, 250417_general_poc_syn

1.3 OCR 실행

# Command 방식 ${docai3 or docai4} data ocr \ -d payment_confirmation \ -n 250213_meritzsec_main \ -ont ont.payment_confirmation.upstage__4.0.yaml \ -ver 2.4.0 \ --dos --language_tag ko

또는 Prompt 방식:

? Select Document Type → 문서 타입 선택
? Select Dataset Name → 데이터셋 선택
? Select ontology → 온톨로지 선택

1.4 (KVS만) 데이터 분석

# KVS 전용 — 토큰 분포 분석 docai4 data analyze

입력/출력 토큰 길이 분포를 분석하여 setup.yaml의 max_length 설정에 활용합니다.

모델 학습
setup.yaml 설정 → 학습 실행

2.1 학습 세팅

# Command 방식 ${docai3 or docai4} model setup \ -d payment_confirmation \ -n 250213_meritzsec_main

또는 Prompt 방식:

? Select Document Type → 문서 타입 선택
? Enter Experiment Name → {날짜}_{고객사}_{범위} 입력

{document_type}/models/{exp_name}/setup.yaml 이 생성됩니다. 아래 주요 항목을 설정하세요:

항목설명예시
data.dataset사용할 데이터셋 이름250417_hi_maintenance
data.ontology온톨로지 파일명ont.diagnosis.upstage__7.0.yaml
train.model모델 타입DRSP: parsing/bio_with_grouping
KVS: lxdonut_parallel
train.lr학습률2e-5
train.batch_size배치 크기2 ~ 4
train.num_epochs에포크 (KVS)200
train.max_steps학습 스텝 (DRSP)1000
train.max_length최대 토큰 길이2048
pretrained_model_...사전학습 모델 경로체크포인트 경로

후처리
Refiner / Grounder 개발 → 재평가
⚠️
현재 /data/project/docai/repos/ocr-parser-postprocessor를 직접 수정하는 방식으로 전환 예정입니다. 전환 전까지는 GitHub PR 방식을 유지합니다.
  1. 개인 공간에 클론:
    $ git clone https://github.com/UpstageAI/ocr-parser-postprocessor
  2. 후처리 개발 후 PR & Merge 완료
  3. 공용 레포 최신화:
    $ cd /data/project/enterprise/docai/repos/ocr-parser-postprocessor && git pull
  4. use_case 지정하여 재평가:
    # 후처리 적용 후 재평가 ${docai3 or docai4} model test \ -d transfer_confirmation \ -u meritzsec \ -n 250729_meritzsec_main \ -e 250729_meritzsec_main__0

    결과: outputs/{use_case}__{ontology_name}/

색상의미

RedWrong — 오답 예측 (fail)
YellowSimilar-but-fail — 유사하지만 불일치

Data Commands

data add

데이터셋 & 온톨로지 복사

/data/project/datasets/data_assets/에서 데이터셋을, /data/project/enterprise/docai/repos/ocr-parser-ontology/_composed/에서 온톨로지를 복사합니다.

Command 방식
${docai3 or docai4} data add \ -d ${document_type} \ -n ${dataset_name} \ -src ${source_path} \ -ver ${source_version} \ -ont ${ontology_name_or_path}
옵션설명예시
-d, --document_type문서 종류-d payment_confirmation
-n, --name데이터셋 이름-n 250213_meritzsec_main
-src, --source복사할 데이터셋 경로-src /data/project/datasets/data_assets/payment_confirmation/drp.en_ko...
-ver, --version데이터셋 릴리즈 날짜-ver 230525
-ont, --ontology온톨로지 이름/경로-ont ont.payment_confirmation.upstage/4.0.yaml
Prompt 방식
? Select Document Type → 문서 타입 선택 (또는 Create New Directory)
? Select Add Type → ontology | source
  ontology → ? Enter ontology path ont.payment_confirmation.upstage/4.0.yaml
  source → ? Enter data source path → ? Select Version → ? Enter dataset name
Dataset Name Convention: {날짜}_{고객사코드}_{범위}_{특징}
날짜: 250213  |  고객사: meritzsec, general  |  범위: main, maintenance, poc  |  특징: syn, _old, _new

data ocr

OCR 실행 & gold 파일 생성
Command 방식
${docai3 or docai4} data ocr \ -d ${document_type} \ -n ${dataset_name} \ -ont ${ontology_name} \ -ver ${ocr_version} \ --dos --is_multipage --language_tag "ko"
옵션설명기본값
-d, --document_type문서 종류—
-n, --name데이터셋 이름—
-ont, --ontology온톨로지 파일명—
-ver, --versionOCR 버전2.2.1
--dosDOS flagTrue
--is_multipage멀티페이지 flagFalse
--language_tag언어 태그"ko"
추가 생성 파일
sources/{dataset_name}/
├── ufos/ocr__{ver}__{lang}__{dos}_dos__edge/  ← OCR 결과
├── ufps/   (train/val/test_ufp.json)

└── csvs/   (train_csv/ val_csv/ test_csv/)

Model Commands

모델 학습, 추론, 평가를 진행하는 명령어입니다.

⚙

model setup

setup.yaml 생성
${docai3 or docai4} model setup \ -d ${document_type} \ -n ${exp_name}

결과: dev/${docai3 or docai4}/{document_type}/models/{exp_name}/setup.yaml

DRSP setup.yaml 주요 항목
  • data.dataset / data.ontology / data.ontology_short
  • train.model: parsing/bio_with_grouping 등
  • backbone_pretrained_model_name_or_path
  • tokenizer_pretrained_model_name_or_path
  • max_length: 2048 / 2400
  • batch_size: 1~4
  • lr: 2e-5~3e-5 / max_steps: 1000~30000
  • aug_methods_to_use: bbox_shake 등
KVS setup.yaml 주요 항목
  • data.dataset / data.ontology
  • train.model: lxdonut_parallel
  • prompt_manager: parallel_with_group / parallel_with_nl_group
  • pretrained_model_name_or_path
  • textual_encoder_max_ocr_tokens: 2048
  • textual_decoder_max_length: 2048
  • num_epochs: 100~200
  • gradient_accumulation_steps: 16
  • precision: fp16
