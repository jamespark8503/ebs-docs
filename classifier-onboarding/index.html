환경 설정
Docker (고객사) / Conda (서버)

Case 1: 고객사 (Docker)

사전 반입 파일:

항목경로
훈련용 Docker/data/project/classifier/version1/docker/train/doccls_train_20250918.tar.gz
추론용 Docker/data/project/classifier/version1/docker/inference/doccls_v1.3.4_general_20251016.tar.gz
학습 코드/data/project/classifier/version1/ocr-document-classifier
# 1. Docker 이미지 로드 docker load < /data/project/classifier/version1/docker/train/doccls_train_20250918.tar.gz # 2. 컨테이너 실행 (ocr-document-classifier 디렉토리에서) cd /data/project/classifier/version1/ocr-document-classifier bash docker_run_for_train.sh # 3. 컨테이너 진입 → /workspace 가 코드 실행 디렉토리 docker exec -it {container_id} /bin/bash
⚠️
docker_run_for_train.sh 내부에서 WORKSPACE="./"로 설정하고, -p 포트를 지정하세요.
이후 모든 작업은 컨테이너 내부 /workspace에서 진행합니다.

Case 2: 회사 서버 (Conda)

# 공용 가상환경 사용 source /data/project/classifier/miniconda3/bin/activate document_classifier # 또는 GitHub에서 clone 후 직접 설치 git clone git@github.com:UpstageAI/ocr-document-classifier.git cd ocr-document-classifier pip install -r requirements/train.txt
💡
처음이라면 /data/project/classifier/version1/ocr-document-classifier에서 바로 진행하는 것을 추천합니다.
코드 실행 위치는 항상 ocr-document-classifier/ 루트입니다.
2
데이터 준비
원본 데이터 → train/val/test 분할 + label

2.1 원본 데이터 구성

클래스별 폴더에 이미지를 넣고, category.yaml을 작성합니다.

dataset/origin_data/          ← IMAGE_PATH
├── category.yaml              ← 클래스 목록
├── medical_bill_receipts/     ← 클래스명 = 폴더명
│   ├── 1.jpg
│   └── 2.jpg
├── diagnosis/
│   └── ...
└── multi/                     ← 멀티 라벨 (특수 폴더)
    └── 1.tmp_2.tmp/          ← _ 로 분리 → .으로 재분리
        └── 1.jpg
# category.yaml 예시 - medical_bill_receipts - diagnosis - medical_expenses_details - merged_medical_certificates
📌
category.yaml의 클래스명은 추론 결과에 직접 매핑됩니다.
나중에 이름을 변경하면 추론 결과도 변경됩니다. (예: medical_bill_receipts → mbr)

2.2 project-build.sh 실행

# scripts/project-build.sh 내부 argument 설정 PROJECT_PATH='dataset/project_data' # 변환 결과 경로 IMAGE_PATH='dataset/origin_data' # 원본 데이터 경로 WORK_DIR='./' # 고정 ROTATION_CLASS='[]' # 비워둠 (pred_rot90 사용) # 실행 → train 80% / val 10% / test 10% 자동 분할 bash scripts/project-build.sh

2.3 결과 구조

dataset/project_data/          ← PROJECT_PATH
├── category.yaml
├── images/
│   ├── train/  val/  test/
└── labels/
    ├── train.txt              ← "이미지명,클래스명" 형식
    ├── val.txt
    └── test.txt
💡
분할 비율 변경: ocr_document_classifier/runner/preprocess_from_dir.py의 train_data_ratio, val_data_ratio, test_data_ratio

모델 학습
Config 설정 → 학습 실행 → 체크포인트

3.1 학습 Config 설정

ocr_document_classifier/configs/train_image.yaml에서 주요 항목:

항목기본값설명
pred_rot90true필수 확인! 회전 감지 활성화 (1장→4장 증강)
defaults.train/modeltrain_efficientnetv2-s모델 선택: -s(small), -m(medium), train_kvs
pl_trainer.max_epochs50학습 에포크 수
accumulate_grad_batches4배치 누적 (실질 batch = 16x4=64)
batch_size16배치 크기 (16x4 조합 권장)
optimizer.lr5e-5학습률 (rot90 사용 시 높게 권장)
gaussnoise.var_limit1.0노이즈 증강 강도 (노이즈 적으면 0.3~0.5)
📌
모델 선택은 결과론적입니다. 항상 -m이 좋은 것은 아닙니다. 여러 모델을 학습 후 정확도가 높은 것을 선택하세요.

3.2 학습 실행

# scripts/train.sh 내부 argument PROJECT_PATH='dataset/project_data' MODEL_VERSION='cls' # 원하는 모델 이름 ocr_document_classifier_path='./' # 실행 bash scripts/train.sh # 백그라운드 실행 nohup bash scripts/train.sh > train.log 2>&1 & tail -f train.log # 로그 확인

패키징 & 서빙 검증
모델 패키지 생성 → Docker 서빙 테스트

5.1 패키징

# scripts/packaging.sh 내부 argument PROJECT_PATH="dataset/project_data" MODEL_VERSION="cls" CHECKPOINT_NAME="epoch=50-val_acc=0.89" # 실행 bash scripts/packaging.sh
dataset/project_data/models/cls/package/
├── category.yaml       ← 클래스 정보
├── model_cfg.yaml      ← 모델 구조 정보
├── model.ckpt          ← 모델 가중치
├── train_config.yaml   ← 학습 설정
├── config.tar.gz       ← config 압축
└── weight.tar.gz       ← 모델 압축 (전달용)

5.2 서빙 검증 (Docker)

# 추론용 Docker로 서빙 테스트 docker run --gpus all \ -p 8080:8080 \ -v $(realpath dataset/project_data/models/cls/package):/app/model_dir \ -e OP_MODEL_DIR=/app/model_dir \ ocr_document_classifier:1.3.0 # 추론 요청 테스트 python tools/kserve/send.py --image_path test.jpg
✅
추론이 정상 동작하면 weight.tar.gz와 config.tar.gz를 고객사에 전달합니다.

FAQ & Troubleshooting
자주 발생하는 오류와 해결 방법
증상원인해결
PermissionError: Permission deniedtorch 캐시 경로 권한 문제/root/.cache/torch → ~/.cache/torch로 변경
RandomResizedCrop ValidationErrorAlbumentations 버전 변경A.RandomResizedCrop(height=H, width=W) →
A.RandomResizedCrop(size=(H, W))
학습이 느림데이터 로딩 병목num_workers: 8, persistent_workers: true 확인
val_acc가 안 오름lr이 너무 낮거나 데이터 부족lr: 5e-5 → 1e-4 시도, 데이터 증강 확인
멀티 라벨 인식 안 됨multi 폴더 구조 오류멀티 클래스: multi/1.tmp_2.tmp/ 형식 (_로 분리 후 .으로 재분리)
